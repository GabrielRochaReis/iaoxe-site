## Por que a maioria dos prompts em português é ruim

O problema é simples: quase todo tutorial de prompt engineering está em inglês, pensado para o contexto americano. Quando você traduz diretamente, o resultado fica artificial — o modelo responde como se estivesse traduzindo do inglês, e você percebe isso na construção das frases.

A solução não é só escrever em português. É escrever prompts que funcionam *na prática* para o contexto brasileiro: mercado local, tom de voz adequado, referências culturais certas, e formatos que fazem sentido para o seu fluxo de trabalho.

Outra coisa que aprendi: prompts vagos geram respostas vagas. Quanto mais específico você for sobre o formato, o tom e o objetivo, mais o modelo entrega algo utilizável sem precisar de 5 rodadas de ajuste.

## Estrutura de um bom prompt (a base de tudo)

Antes da lista, entenda o padrão que uso em praticamente todos os prompts eficazes:

```
[Papel] + [Contexto] + [Tarefa] + [Formato] + [Restrições]
```

**Exemplo ruim:**
> "Escreve um e-mail para cliente"

**Exemplo bom:**
> "Você é um gerente de contas de software B2B. Escreva um e-mail de follow-up para um cliente que testou nosso produto por 14 dias e não renovou. Tom profissional mas caloroso, sem pressão de venda. Máximo 150 palavras. Não use expressões como 'espero que esteja bem' ou 'qualquer dúvida estou à disposição'."

A diferença no resultado é absurda. Agora, a lista.

## Prompts para trabalho e produtividade

### Resumir reunião / ata

```
Transforme o seguinte texto em uma ata de reunião clara. Estruture em: Presentes (deixe em branco), Decisões tomadas, Próximos passos com responsável e prazo. Tom objetivo, frases curtas. Texto: [cole aqui]
```

### Escrever e-mail profissional difícil

```
Escreva um e-mail profissional em português brasileiro para [situação: ex: cobrar prazo de entrega de fornecedor atrasado]. Tom assertivo mas respeitoso. Evite clichês. O remetente é [seu cargo]. Máximo 120 palavras.
```

### Transformar rascunho em texto polido

```
Reescreva o texto abaixo mantendo todas as informações. Melhore a clareza, elimine redundâncias, ajuste o tom para comunicação corporativa formal. Não adicione informações que não estão no original. Texto: [cole aqui]
```

### Criar agenda de reunião

```
Crie uma agenda para uma reunião de [duração] sobre [tema]. Inclua: objetivo da reunião, tópicos com tempo estimado, espaço para perguntas. Formato de lista numerada, linguagem objetiva.
```

### Estruturar relatório executivo

```
Você é um analista de negócios sênior. Com base nos dados abaixo, escreva um relatório executivo de 1 página. Estrutura: Resumo Executivo (3 linhas), Principais Achados (bullet points), Recomendação (1 parágrafo). Dados: [cole aqui]
```

## Prompts para marketing e conteúdo

### Criar legenda para Instagram

```
Crie 3 opções de legenda para Instagram sobre [tema/produto]. Público-alvo: [descreva]. Tom: [ex: descontraído, informativo, inspirador]. Inclua emojis relevantes e um CTA no final. Máximo 150 caracteres por opção.

### Gerar ideias de posts

```
Liste 15 ideias de posts para [nicho] no Instagram/LinkedIn para o mês de [mês]. Foque em conteúdo educativo e conteúdo que gera salvamentos. Formato: título do post + tipo (carrossel, reels, imagem estática).

## Prompts para programadores

### Revisar e explicar código

```
Revise o código abaixo e: 1) identifique bugs ou problemas de performance, 2) explique em português o que cada parte faz, 3) sugira melhorias com justificativa. Linguagem: [Java/Python/JS/etc]. Código: [cole aqui]
```

### Converter código entre linguagens

```
Converta o código abaixo de [linguagem origem] para [linguagem destino]. Mantenha a mesma lógica. Adicione comentários em português explicando trechos não óbvios. Código: [cole aqui]
```

### Escrever testes unitários

```
Escreva testes unitários para o método abaixo usando [JUnit/Jest/Pytest]. Cubra: caso feliz, caso de erro, edge cases (null, lista vazia, limite de caracteres). Código: [cole aqui]
```

### Criar documentação técnica

```
Com base no código abaixo, escreva documentação técnica em português no formato Markdown. Inclua: descrição do módulo, parâmetros de entrada/saída, exemplos de uso, possíveis erros. Código: [cole aqui]
```

### Debug guiado

```
Estou recebendo o seguinte erro: [cole o erro]. Stack trace: [cole aqui]. Contexto: [descreva o que estava fazendo]. Me ajude a: 1) entender a causa raiz, 2) identificar onde no código buscar, 3) sugerir a correção mais simples.
```

## Prompts para estudo e aprendizado

### Explicar conceito difícil de forma simples

```
Explique [conceito] como se eu fosse um estudante de tecnologia sem experiência no assunto. Use analogias do dia a dia, evite jargão técnico no início. Depois apresente a versão técnica correta. No máximo 300 palavras.
```

### Criar plano de estudos

```
Crie um plano de estudos de [duração] para aprender [tema]. Sou [nível: iniciante/intermediário/avançado] e tenho [X horas/semana] disponíveis. Inclua: recursos gratuitos em português quando possível, ordem lógica de aprendizado, marcos de progresso verificáveis.
```

### Preparar para entrevista técnica

```
Simule uma entrevista técnica para a vaga de [cargo]. Faça 5 perguntas progressivamente mais difíceis, em português, sobre [tecnologias/temas]. Após cada resposta minha, dê feedback honesto sobre o que faltou ou poderia ser mais preciso.
```

## Prompts para análise e decisão

### Analisar prós e contras

```
Preciso decidir entre [opção A] e [opção B] para [contexto]. Liste prós e contras de cada um, considerando: [critérios importantes para mim]. No final, dê uma recomendação clara com justificativa. Não fique em cima do muro.
```

### Analisar documento / contrato

```
Analise o documento abaixo e me informe: 1) pontos principais em bullet points, 2) cláusulas que merecem atenção especial ou que são incomuns, 3) o que está faltando ou poderia ser um problema. Seja objetivo e use linguagem acessível. Documento: [cole aqui]
```

### Criar plano de ação

```
Transforme o objetivo abaixo em um plano de ação concreto. Quebre em tarefas específicas, com sequência lógica, estimativa de tempo para cada tarefa e o que preciso ter/saber antes de começar. Objetivo: [descreva]. Prazo: [data].
```

## Erros comuns ao usar prompts em português

**1. Prompt muito curto e vago**
"Me ajuda a escrever um texto" não diz nada. Para onde vai o texto? Para quem? Qual tom? Qual tamanho?

**2. Não especificar o formato da resposta**
Se você quer uma lista, diga "em formato de lista". Se quer parágrafos, diga "em texto corrido com parágrafos". O modelo responde no formato que inferir do contexto, e nem sempre infere certo.

**3. Não dar contexto suficiente**
"Escreva um e-mail para meu cliente" é diferente de "Escreva um e-mail para uma empresa de médio porte do setor financeiro que usa nosso software de RH há 6 meses". O segundo gera resultado 10x melhor.

**4. Aceitar o primeiro resultado sem iterar**
O ChatGPT responde bem à instrução "refaça isso, mas com tom mais direto" ou "muito longo, corte pela metade mantendo os pontos principais". A maioria das pessoas não itera e fica com um resultado mediano.

**5. Usar prompts em inglês traduzidos literalmente**
"You are a helpful assistant that..." fica estranho em português e o modelo tende a responder parcialmente em inglês. Escreva seus prompts inteiramente em português quando quiser respostas em português.

**6. Não dizer o que *não* quer**
Adicionar restrições melhora muito o resultado: "não use clichês", "não repita palavras", "não inclua disclaimers legais", "não use bullet points".

## Perguntas frequentes

**Prompts em inglês dão resultados melhores?**
Para tarefas técnicas (código, análise de dados) sim, levemente. Para texto em português ou contexto brasileiro, escrever em português é melhor — o modelo calibra o tom e o vocabulário de forma mais natural.

**Posso usar os mesmos prompts no Claude?**
Sim, a maioria funciona nos dois. O Claude tende a ser mais literal nas instruções de formato, então especifique bem o que quer. Para texto longo em português, Claude costuma entregar resultado mais fluido.

**Existe um tamanho ideal de prompt?**
Não existe regra fixa. Prompts mais curtos são mais rápidos, mas prompts mais detalhados (para tarefas complexas) geram resultados mais precisos. Para tarefas simples, 2-4 linhas bastam. Para tarefas críticas, 10-15 linhas de contexto fazem diferença.

**Como guardar e reutilizar meus melhores prompts?**
No ChatGPT você pode usar "Custom Instructions" ou criar GPTs personalizados. Eu uso um arquivo de texto simples no Obsidian mesmo — funciona para qualquer ferramenta. Ferramentas como o [**Rytr**](https://rytr.me?via=gabriel-reis) já têm templates prontos por caso de uso, o que economiza tempo quando você precisa de algo padrão (e-mail, blog post, bio) sem reinventar a roda.

**O ChatGPT "esquece" meus prompts entre conversas. Como resolver?**
Crie um documento modelo com seu contexto padrão e cole no início de conversas novas. Para contextos frequentes, o recurso "Projects" do Claude ou "Memory" do ChatGPT ajudam a persistir informações básicas sem precisar repetir.
